========================================
  语音助手 Cloudflare Worker 部署包
========================================

【ZIP 内容】
  worker/
    src/index.js      — Worker 主程序
    wrangler.toml     — Cloudflare 配置
    README.md         — 英文说明（可选阅读）

【前置条件】
  1. 已注册 Cloudflare 账号（免费即可）
  2. 本机已安装 Node.js 20+
  3. 已准备好 OpenAI API Key（sk- 开头）

【部署步骤（约 10 分钟）】

  1. 解压 ZIP，进入 worker 目录

  2. 安装 Wrangler 并登录
     npm install -g wrangler
     wrangler login

  3. 设置 Secret（见下方「环境变量说明」）
     wrangler secret put OPENAI_API_KEY
     （按提示粘贴您的 OpenAI Key，输入时不可见，属正常）

  4. 部署
     wrangler deploy

  5. 终端会输出地址，例如：
     https://voice-ai-proxy.xxx.workers.dev

  6. 验证（浏览器或 curl）
     https://您的地址/health
     应看到：{"status":"ok"}

  7. 把完整的 workers.dev 地址发给我们（不要末尾斜杠）

========================================
  Secret / 环境变量说明
========================================

【必填 Secret — 仅通过 wrangler secret put 设置，勿写入文件】

  名称              说明
  ----------------  ------------------------------------------
  OPENAI_API_KEY    您的 OpenAI API Key（sk-...）

【可选 Secret — 防滥用门禁，MVP 可不设】

  名称              说明
  ----------------  ------------------------------------------
  PROXY_AUTH_TOKEN  自定义一串随机密码；若设置，须告知我们，
                    以便写入 APK 构建配置（VITE_PROXY_TOKEN）

【已在 wrangler.toml 中的普通变量 — 一般无需改】

  名称              默认值          说明
  ----------------  --------------  --------------------------
  OPENAI_MODEL      gpt-5.6-terra   OpenAI 模型（聪明与费用平衡）
  RATE_LIMIT_MAX    60              限流上限（需配 KV 才生效）

【不要设置】
  - 不要把 OPENAI_API_KEY 写进 wrangler.toml 或任何源码
  - 不要把 Key 提交到 Git

========================================
  部署成功后请回复我们
========================================

  1. Worker 公网地址（例：https://voice-ai-proxy.xxx.workers.dev）
  2. 是否设置了 PROXY_AUTH_TOKEN（若设置了，请私下告知 Token 值）
  3. /health 是否返回 ok

  收到地址后，我们将：
  - 把地址写入 APK 构建配置
  - 重新打包发给您的用户「装完即用」
